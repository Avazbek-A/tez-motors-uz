# Tez Motors — Keys & Secrets: what each one is and how to get it

Every key goes into **`.env.local`** on the Vostro (the bootstrap creates the
file from `.env.example`). Edit with `nano .env.local`, one `KEY=value` per line,
no quotes needed, then `sudo systemctl restart tez-motors` (or re-run
`bash deploy/selfhost/bootstrap.sh`).

**Golden rule:** the platform is **fail-open**. Only the 4 keys in Tier 0 are
required. Every other key turns a feature ON; leave it blank and that feature
simply stays off — nothing crashes. So you can launch with Tier 0 + Tier 1 today
and add the rest whenever you sign up for each provider.

> ⚠️ `NEXT_PUBLIC_*` keys are **baked into the build**. If you change one, you must
> rebuild (`bash deploy/selfhost/bootstrap.sh`), not just restart. All other keys
> are read at runtime, so a restart is enough.

## Tier 0 — Required (the site won't build without these)

### Supabase keys
The database + storage. One free project powers everything.

1. Go to **https://supabase.com** → sign in → open your project (or **New project**).
2. Left sidebar → **Project Settings** (gear) → **API**.
3. Copy:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL` (looks like `https://abcd1234.supabase.co`)
   - **Project API keys → `anon` `public`** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **Project API keys → `service_role` `secret`** → `SUPABASE_SERVICE_ROLE_KEY`
4. **Database connection string** (for backups later): Project Settings → **Database** → **Connection string → URI** → that's your `SUPABASE_DB_URL` (used by `pg_dump`, not stored in `.env.local`).

> 🔒 The `service_role` key bypasses all security rules — it's server-only. Never
> put it in a `NEXT_PUBLIC_*` var, never expose it to a browser.

### ADMIN_PASSWORD
You invent this. It's the password for **https://tezmotors.uz/admin**.
- Make it long and random. Generate one: `openssl rand -base64 24`.
- After handover, **rotate it** and give the client the new value via 1Password or
  a signed PDF — never over Telegram/SMS.

### NEXT_PUBLIC_SITE_URL
Your public address. Set to `https://tezmotors.uz` (or `http://localhost:3000`
while testing). Used for absolute links, emails, sitemaps.

## Tier 1 — Secrets you generate yourself (no signup)

These are shared passwords between two parts of *your own* system. Generate each
with **`openssl rand -hex 32`** and paste the result. (The bootstrap already
auto-generates `UPDATE_WEBHOOK_SECRET` for you.)

```bash
openssl rand -hex 32     # run once per secret, paste the output
```

| Key | What it protects |
|---|---|
| `UPDATE_WEBHOOK_SECRET` | The `git push → deploy` webhook. Same value goes into GitHub (Deploy guide §D). **Auto-generated by bootstrap.** |
| `CRON_SECRET` | The `/api/cron/*` scheduled jobs. Same value used by your crontab / the cron-worker. |
| `MARKET_INGEST_SECRET` | The OLX/Telegram price collectors that POST data to the app. |
| `EXTRACTOR_SECRET` | Optional bearer between the app and the local AutoHome extractor service. |
| `TELEGRAM_WEBHOOK_SECRET` | Validates that inbound Telegram webhook calls really come from Telegram. |
| `WHATSAPP_VERIFY_TOKEN` | A value *you choose*; you paste the same into Meta's webhook setup. |

### VAPID keys (Web Push)
For browser push notifications. Generate a keypair once:
```bash
npx web-push generate-vapid-keys
```
- `Public Key` → `VAPID_PUBLIC_KEY` **and** `NEXT_PUBLIC_VAPID_PUBLIC_KEY` (same value, both vars)
- `Private Key` → `VAPID_PRIVATE_KEY`
- `VAPID_SUBJECT` → `mailto:you@tezmotors.uz`

## Tier 2 — Strongly recommended

### Telegram bot — lead alerts, deploy ✅/❌ pings, downtime alerts
1. In Telegram, open **@BotFather** → send `/newbot` → follow prompts → it gives you
   a **token** like `7123456789:AAH...` → that's `TELEGRAM_BOT_TOKEN`.
2. Get your chat id: message **@userinfobot** (it replies with your numeric id), or
   send any message to your bot then open
   `https://api.telegram.org/bot<TOKEN>/getUpdates` and read `chat.id`.
   - Personal alerts → put your id in `TELEGRAM_CHAT_ID`.
   - `TELEGRAM_ERROR_CHAT_ID` (optional) → a separate chat/channel for ops alerts.
   - `TELEGRAM_CHANNEL_ID` (optional) → a public channel like `@tezmotors` to
     auto-post new cars (add the bot as an admin of that channel first).
   - `TELEGRAM_OPERATOR_CHAT_IDS` (optional) → your id(s), comma-separated, to run
     the **Dealer Copilot** from Telegram.
3. To receive inbound bot messages, also set `TELEGRAM_WEBHOOK_SECRET` (Tier 1)
   and register the webhook once (command is in `.env.example`).

### Resend — transactional email
1. Sign up at **https://resend.com** (free tier).
2. **API Keys** → **Create API Key** → copy → `RESEND_API_KEY`.
3. **Domains** → **Add Domain** (`tezmotors.uz`) → add the shown **SPF/DKIM DNS
   records** in Cloudflare → wait for "Verified".
4. `EMAIL_FROM` → a sender on that verified domain, e.g.
   `Tez Motors <noreply@tezmotors.uz>`.
5. `DEALER_EMAIL` → the inbox that should receive new-lead alerts.

> Until the domain is verified, leave `RESEND_API_KEY` blank — the app falls back
> to Telegram-only notifications.

## Tier 3 — Payments (Uzbekistan)

Both ship **dark** (the pay button is hidden) until you provide keys, so there's
zero risk before onboarding.

### Payme (Paycom)
1. Apply for a merchant account at **https://payme.uz** (business onboarding).
2. In the Payme **Merchant Cabinet**, create a cashbox/merchant; you receive:
   - **Merchant ID** → `PAYME_MERCHANT_ID` **and** `NEXT_PUBLIC_PAYME_MERCHANT_ID`
   - **Key** (the cashbox key used to verify callbacks) → `PAYME_MERCHANT_KEY`
3. Set the **Endpoint** in the cabinet to `https://tezmotors.uz/api/payments/payme`.
4. Pass their sandbox test suite, then switch to production keys.

### Click
1. Apply at **https://click.uz** for a merchant/service.
2. You receive: **Service ID** → `CLICK_SERVICE_ID`, **Merchant ID** →
   `CLICK_MERCHANT_ID` **and** `NEXT_PUBLIC_CLICK_MERCHANT_ID`, **Secret Key** →
   `CLICK_SECRET_KEY`.
3. Set the Prepare/Complete callback URL to `https://tezmotors.uz/api/payments/click`.

## Tier 4 — Customer engagement (optional)

### Eskiz.uz — SMS OTP (phone login)
1. Register at **https://eskiz.uz** (SMS provider for Uzbekistan).
2. Use your account login → `ESKIZ_EMAIL` + `ESKIZ_PASSWORD`.
3. Submit a **sender alphaname** for approval; once approved put it in `ESKIZ_FROM`
   (defaults to Eskiz's `4546` test sender if unset).
- Until set, OTP codes are written to the server log so phone login still works in dev.

### WhatsApp Business (Meta Cloud API)
1. **https://developers.facebook.com** → create an app → add the **WhatsApp** product.
2. From the WhatsApp setup screen:
   - **Temporary/Permanent access token** → `WHATSAPP_TOKEN` (use a permanent
     System User token for production)
   - **Phone number ID** → `WHATSAPP_PHONE_ID`
   - **App secret** (App → Settings → Basic) → `WHATSAPP_APP_SECRET`
3. `WHATSAPP_VERIFY_TOKEN` → any value you choose (Tier 1); enter the same in the
   webhook config.
4. Set the webhook callback to `https://tezmotors.uz/api/bot/whatsapp`, verify, and
   subscribe to the **messages** field.

### Web Push — see VAPID keys in Tier 1.

## Tier 5 — AI (pick ONE)

### Option A (recommended on the Vostro): free local Ollama — no key
Runs on the same machine, $0 per token. See Deploy guide §F:
```
LLM_PROVIDER=openai
LLM_API_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5:7b-instruct
LLM_VISION_MODEL=qwen2.5-vl
# LLM_API_KEY=   (leave empty)
```

### Option B: hosted (Anthropic / OpenAI-compatible)
- Anthropic: **https://console.anthropic.com** → API Keys → `LLM_API_KEY` (leave
  `LLM_PROVIDER`/`LLM_API_URL` blank to default to Anthropic).
- Any OpenAI-compatible (Groq/Together/OpenAI): set `LLM_PROVIDER=openai`,
  `LLM_API_URL=<their /v1 URL>`, `LLM_API_KEY=<key>`, `LLM_MODEL=<model>`.

`AI_AUTORESPOND=1` (optional) auto-sends an AI-drafted reply to leads that leave
an email. Leave blank for a standard confirmation only.

## Tier 6 — Anti-bot (Cloudflare Turnstile)
Invisible spam protection on public forms.
1. Cloudflare dashboard → **Turnstile** → **Add widget** → add `tezmotors.uz`.
2. Copy **Site Key** → `NEXT_PUBLIC_TURNSTILE_SITE_KEY`, **Secret Key** →
   `TURNSTILE_SECRET`.
- Unset = forms still work (honeypot + rate limiter remain the gate).

## Tier 7 — Analytics & search consoles (optional)

| Key | Where |
|---|---|
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | **plausible.io** → add site → use your domain (`tezmotors.uz`). Privacy-friendly traffic stats. |
| `NEXT_PUBLIC_TAWK_ID` | **tawk.to** → create a property → Admin → Channels → Chat Widget → the widget id in the embed (`propertyId/widgetId`). Live human chat. |
| `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION` | **Google Search Console** → add property → "HTML tag" method → copy the `content="..."` value. |
| `NEXT_PUBLIC_BING_VERIFICATION` | **Bing Webmaster Tools** → add site → meta-tag method → the `content` value. |
| `NEXT_PUBLIC_YANDEX_VERIFICATION` | **Yandex Webmaster** → add site → meta-tag method → the `content` value. |

(All `NEXT_PUBLIC_*` → require a rebuild after changing.)

## Tier 8 — Deploy (set by the bootstrap)
- `UPDATE_WEBHOOK_SECRET` — auto-generated (Tier 1); paste into GitHub (Deploy §D).
- `DEPLOY_BRANCH` — set to `main` by the bootstrap; change only if you deploy a
  different branch.

## The fastest launch path

To go live **today** you only need:
1. **Tier 0** — the 4 Supabase/admin keys.
2. **Tier 1** — let the bootstrap generate `UPDATE_WEBHOOK_SECRET`; generate
   `CRON_SECRET` if you'll run scheduled jobs.
3. **Tier 2 (Telegram bot)** — 2 minutes, and you instantly get lead alerts +
   deploy pings + downtime warnings.

Add Resend, payments, SMS, push, WhatsApp, AI, analytics later — each is a blank
line today and a working feature the moment you fill it. Nothing breaks in the
meantime.

## Worked example of a minimal `.env.local`

```
# Tier 0 (required)
NEXT_PUBLIC_SUPABASE_URL=https://abcd1234.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOi...
ADMIN_PASSWORD=a-long-random-password
NEXT_PUBLIC_SITE_URL=https://tezmotors.uz

# Tier 1 (generated: openssl rand -hex 32)
UPDATE_WEBHOOK_SECRET=3f9c...           # bootstrap fills this
CRON_SECRET=7a2b...

# Tier 2 (recommended — Telegram)
TELEGRAM_BOT_TOKEN=7123456789:AAH...
TELEGRAM_CHAT_ID=123456789

# Free AI (Ollama)
LLM_PROVIDER=openai
LLM_API_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5:7b-instruct
```
Everything else can stay blank until you need it.
